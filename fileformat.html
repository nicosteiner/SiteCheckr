SiteCheckr-rulesets are saved as XML.

Meta-Data

General information about the ruleset (title, description, author). Can be displayed in future versions of SiteCheckr.

{% highlight xml %} Title Description author@example.org {% endhighlight %}

Rules

Rules are placed in a <rules>-node.

A single rule has the following structure:

{% highlight xml %} Title {% endhighlight %}
priority

optional, defaults to: required

required
Rule must be fulfilled
nicetohave
Rule should be fulfilled, but it is not necessary to get a green SiteCheckr icon.
mode

optional, defaults to: required

required
Test must pass.
forbidden
Test has to fail.
type

optional, defaults to: xpath

xpath
Test is a XPath expression
css
Test is a CSS selector
